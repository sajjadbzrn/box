# بنچمارک — عملکرد

Box از یک بنچمارک داخلی برای مقایسه توان عملیاتی با Hono استفاده می‌کند.

## اجرای بنچمارک

```bash
bun run packages/benchmark/src/run-bench.ts
```

## نتایج فعلی (200,000 درخواست)

| مسیر | Box (req/s) | Hono (req/s) | نسبت |
|------|-------------|--------------|-------|
| ایستا `GET /` | ~2,000,000 | ~3,000,000 | ~66% |
| پارامتری `GET /user/:id` | ~1,600,000 | ~2,000,000 | ~80% |

## طراحی عملکرد

Box برای **عملکرد برنامه‌های واقعی** طراحی شده، نه میکرو-بنچمارک:

| ویژگی | هزینه | دلیل |
|--------|-------|------|
| درخت رادیکس | O(طول_مسیر) | به اندازه کافی سریع |
| شی Context | ~1μs | یک بار در هر درخواست |
| پایپلاین میان‌افزار | ~0.5μs به ازای هر لایه | خطی |
| اعتبارسنجی Zod | ~5–50μs | فقط وقتی استفاده کنید |

## نکات بهینه‌سازی

1. از **مسیرهای ایستا** استفاده کنید — سریع‌تر از `:param`
2. **میان‌افزار را به حداقل برسانید** — هر لایه ~0.5μs سربار دارد
3. **اعتبارسنجی را حذف کنید** در مسیرهای داخلی با توان بالا اگر ورودی قابل اعتماد است
