# Router — مسیریابی درخت رادیکس

Box از **درخت رادیکس** (Patricia trie) برای تطبیق مسیر استفاده می‌کند — یک درخت پیشوندی فشرده که جستجوی O(طول_مسیر) با حافظه کم ارائه می‌دهد.

## الگوهای مسیر

```ts
// ایستا — تطابق دقیق
app.get("/users/list", handler);

// پویا — بخش پارامتر به عنوان params
app.get("/user/:id", (c) => {
  c.params.id; // "42"
});

// Wildcard — دریافت مسیر باقی‌مانده
app.get("/static/*", (c) => {
  c.params["*"]; // "js/app.js"
});
```

## اولویت مسیر

مسیرهای ایستا نسبت به مسیرهای پویا اولویت دارند:

```ts
app.get("/version", (c) => c.text("1.0"));   // ایستا — برنده می‌شود
app.get("/:slug", (c) => c.text("پویا"));     // پویا
```

درخواست به `/version` → مسیر ایستا تطبیق می‌خورد.  
درخواست به `/anything-else` → مسیر پویا تطبیق می‌خورد.

## جداسازی متدها

هر متد HTTP درخت رادیکس مخصوص خود را دارد:

```ts
app.get("/item", getItem);
app.post("/item", createItem);
app.put("/item", updateItem);
app.delete("/item", deleteItem);
```

## فرآیند جستجو

1. استخراج متد و مسیر از درخواست
2. انتخاب درخت رادیکس برای آن متد
3. پیمایش درخت با تطبیق بخش‌ها:
   - ابتدا فرزندان **ایستا** (تطبیق دقیق پیشوند)
   - سپس فرزند **پویا** `:param` (ذخیره مقدار بخش)
   - سپس فرزند **wildcard** `*` (ذخیره مسیر باقی‌مانده)
4. بازگرداندن `{ handler, params }` یا `null`
